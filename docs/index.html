Introduction

Getting to know Fries

Fries is an awesome mobile prototyping framework for Android apps inspired by Ratchet. We all know that you can find loads of iOS prototyping tools out there, so this time let's give some love to Android.

Getting started

What to do next after downloading

  1. Create your pages

    Creating the markup for your pages is really easy. Just follow these docs on how to write them. You may also customize the styles to make the prototype your own.

  2. Stitch them up with push.js

    Similar to Ratchet, your pages should be pieced together using push.js. Push.js allows you to create interactive prototypes by dynamically loading HTML on the fly and inserting it to your prototype. Read more about push.js to understand how this works.

  3. Test them on your device

    To test your prototype on your device, serve it up on your local web server and load it up on your Android browser. You can also do this by installing a web server on your Android device and serving the files from there.

Components

UI building blocks

Action bar

Action bars serve as your main navigation throughout your app. These hold the app icon, app title, action buttons, and can also hold tabs.

App icons must be between 24px and 32px in height. It can have any width though as long as your prototype doesn't break its layout.

<header class="action-bar">
  <a href="#" class="app-icon action">
    <i class="icon-home"></i>
  </a>
  <h1 class="title">Title</h1>
</header>

Action bar with Up button

Adding the Up button means that the current screen is not at the top level.

<header class="action-bar">
  <a href="#" class="app-icon action up">
    <i class="chevron"></i>
    <i class="icon-home"></i>
  </a>
  <h1 class="title">Title</h1>
</header>

Action bar with subtitles

You can also create an action bar with a title and a subtitle

<header class="action-bar">
  <a href="#" class="app-icon action">
    <i class="icon-home"></i>
  </a>
  <h2 class="title">Title <span class="subtitle">Subtitle</span></h2>
</header>

Action buttons

Action bars can also have buttons in them.

<header class="action-bar fixed-top">
  <a href="#" class="app-icon action">
    <i class="icon-home"></i>
  </a>
  <h1 class="title">Action Buttons</h1>
  <ul class="actions pull-right">
    <li><a href="#" class="action" title="Search"><i class="icon-search"></i></a></li>
    <li><a href="#" class="action" title="Settings"><i class="icon-settings"></i></a></li>
  </ul>
</header>

Action overflows

You can only add a maximum of two (2) action buttons to the action bar when the device width is less than 480 pixels. This is where action overflows come in.

Action overflows automatically convert action buttons into a Spinner when there are more than two buttons in your action bar. Fries will automatically use the value of the button's title attribute to generate a Spinner item. So make sure that you add a title to your action buttons.

<header class="action-bar fixed-top">
  <a href="#" class="app-icon action">
    <i class="icon-home"></i>
  </a>
  <h1 class="title">Action Buttons</h1>
  <ul class="actions pull-right">
    <li><a href="#" class="action" title="Search"><i class="icon-search"></i></a></li>
    <li><a href="#" class="action" title="Copy"><i class="icon-copy"></i></a></li>
    <li><a href="#" class="action" title="Cut"><i class="icon-scissors"></i></a></li>
    <li><a href="#" class="action" title="Settings"><i class="icon-settings"></i></a></li>
  </ul>
</header>

Tabs

Tabs are usually used to navigate between multiple screens.

<nav class="tab-fixed">
  <ul class="tab-inner">
    <li class="active"><a href="#item1">Tab 1</a></li>
    <li><a href="#item2">Tab 2</a></li>
    <li><a href="#item3">Tab 3</a></li>
  </ul>
</nav>

Buttons

Buttons come in two sizes and supports all button states.

<div class="inset content">
  <a href="#" class="button-normal">Normal</a>
  <a href="#" class="button-small">Small</a>
  <a href="#" class="button-normal focused">Focused Normal</a>
  <a href="#" class="button-small focused">Focused Small</a>
  <a href="#" class="button-normal active">Pressed Normal</a>
  <a href="#" class="button-small active">Pressed Small</a>
  <a href="#" class="button-normal focused disabled">Focused Disabled</a>
  <a href="#" class="button-small focused disabled">Focused Disabled Small</a>
  <a href="#" class="button-normal disabled">Disabled</a>
  <a href="#" class="button-small disabled">Disabled Small</a>
</div>

iScroll

Usage and notes

Fries uses iScroll 4 for its scroll bars and it is really easy to use.

Just add the "scroll" id to the container that you want to attach iScroll to and voila, a sexy scroll bar in your prototype.

// Attaches iScroll to element with id "scroll"
scroller = new iScroll('scroll', {
  fadeScrollbar: true,
  hideScrollbar: true
});

Using iScroll in lists

You must wrap your lists in <div id="scroll" class="content"> to attach iScroll to it.

<div id="scroll" class="content">
  <ul class="list inset">
    <li class="list-divider">Single Line List</li>

Page setup

Oh yes, I copied this from Ratchet. I'm so evil.

Since Fries is purely based on Ratchet, the page setup goes the same.

  1. Fixed bars come first

    All fixed bars and tabs (.action-bar, .tab-fixed) should always be the first thing in the <body> of the page. This is really important!

  2. Everything else goes in .content

    Anything that's not an .action-bar or a .tab-fixed should be put in a div with the class "content". Put this div after the bars in the <body> tag. The .content div is what actually scrolls in a Fries prototype.

  3. Don't forget your meta tags

    The meta tags are included in the template.html that comes with your Fries. Make sure they're in your HTML so Fries works.

<div class="splash">
  <h1>Thank you!</h1>
  <p>Check back again for updates</p>
</div>
